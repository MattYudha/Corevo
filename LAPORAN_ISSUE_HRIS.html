Penjelasan Inti (Analisis Teknikal)

1. Timezone (Kritikal)

  • Sistem default ke UTC karena tidak ada APP_TIMEZONE di .env.
  • Semua record waktu di database mundur 7 jam dari WIB.
Dampak: Seluruh data absensi karyawan tidak valid.

4. Session Logout Bermasalah (Kritikal)

  • Session masih aktif di beberapa device setelah logout.
  • Middleware EnsureRoleSession mengisi ulang session data secara otomatis.
Dampak: Potensi akses ilegal setelah user keluar sistem.

7. Finance & 8. Payroll (Kritikal)

  • Role Finance tidak bisa akses Ledger meski route sudah diset.
  • Menu Payroll hilang total dari navigasi utama (Sidebar).
Dampak: Operasional keuangan dan penggajian terhenti.

12. Logic Late Salah (WFH/WFA)

  • Sistem tidak membedakan jam kantor dengan jam kerja remote.
  • Rule 08:00 tetap diterapkan pada karyawan WFH/WFA.
Dampak: Karyawan remote mendapatkan penalti keterlambatan yang salah.

Rekomendasi Prioritas (Action Plan)

Prioritas Tinggi (Sesi 1)
  • Sinkronisasi Timezone ke Asia/Jakarta
  • Registrasi Menu Payroll di Sidebar Admin
  • Perbaikan Logic Logout & Session Flush
  • Validasi Role Finance & DB Constants
Prioritas Menengah (Sesi 2)
  • Implementasi Field Lembur di Tabel Presence
  • Pembuatan Form Manual Input KPI untuk Admin
  • Update Logic Late Badge (Exclude WFH/WFA)
  • Custom Throttle Limit untuk Admin Bulk Action
Improvement (Sesi 3)
  • Fitur Cleanup/Purge Audit Log Manual
  • Implementasi Sidebar Collapsible Desktop
