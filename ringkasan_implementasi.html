Sesi 1 — Infrastruktur & Konfigurasi Dasar
# Tipe Yang Dikerjakan Penjelasan File
1 FIX Docker container crash saat startup Container lama tidak kompatibel dengan Docker Engine terbaru. Dihapus dan dibuat ulang dari awal. docker-compose.yml
2 FIX Error SSL saat koneksi ke database MySQL SSL certificate verification dinonaktifkan agar koneksi MySQL di Docker tidak crash. config/database.php
3 FIX Timezone server tidak sesuai WIB Ditambahkan APP_TIMEZONE=Asia/Jakarta agar waktu absensi dan log sesuai Waktu Indonesia Barat. .env
4 FIX Logout tidak membersihkan data sesi role Saat logout, data role dan employee_id tersisa di sesi — bisa "bocor" ke pengguna lain. Sekarang sesi dibersihkan total saat logout. AuthenticatedSessionController.php
5 BARU Akses modul Keuangan untuk role Finance Role Finance sebelumnya tidak bisa akses halaman Entities & Accounts. Sekarang sudah diberikan izin di route. routes/web_finance.php
6 BARU Menu Payrolls ditambahkan di sidebar Grup sidebar "Key Performance (KPI)" diubah menjadi "Payroll & KPI" dan menu Payrolls ditambahkan di dalamnya. layouts/dashboard.blade.php
7 FIX Nama role "Super Admin" vs "Master Admin" tidak cocok Database menyimpan "Super Admin" tapi kode menggunakan konstanta "Master Admin" — menyebabkan menu admin tidak tampil. Nama di database diselaraskan. Database (tabel roles)
Sesi 2 — Perbaikan Logika Bisnis (Functional Bugs)
# Tipe Yang Dikerjakan Penjelasan File
1 FIX Badge "Terlambat" muncul di absensi WFH & WFA Karyawan yang bekerja dari rumah (WFH) atau dari mana saja (WFA) tetap mendapat label "Terlambat" meskipun tidak ada aturan waktu untuk mereka. Badge Late sekarang hanya muncul untuk WFO (kerja di kantor). PresencesController.php
2 FIX Aplikasi crash (error 500) saat akses Task ID tidak valid Membuka URL seperti /tasks/999/done padahal task ID 999 tidak ada menyebabkan aplikasi crash dengan halaman error server. Sekarang menampilkan halaman 404 (Tidak Ditemukan) yang proper. TaskController.php
3 FIX Status task bisa diisi sembarang teks Kolom status task tidak divalidasi sehingga bisa diisi teks apapun. Sekarang hanya menerima tiga nilai: pending, on progress, atau done. TaskController.php
4 FIX Pembuat task tidak tersimpan di database Field created_by (siapa yang membuat task) tidak pernah terisi karena tidak diambil dari sesi pengguna yang login. Sekarang otomatis tersimpan. TaskController.php, Task.php
5 FIX Karyawan mendapat error 429 saat absen Batas request saat absen terlalu ketat (hanya 10 per menit). Jika jaringan lambat dan browser mencoba ulang, karyawan langsung diblokir. Dinaikkan menjadi 60 per menit. routes/web.php
6 FIX Admin terkena rate limit saat generate payroll massal Proses generate payroll untuk banyak karyawan sekaligus bisa terpotong oleh batas request default. Route payroll dan KPI recalculate dinaikkan batas menjadi 300 request per menit. routes/web.php
Sesi 3 — Fitur Baru & Peningkatan UX
# Tipe Yang Dikerjakan Penjelasan File
1 BARU Admin bisa edit nilai KPI karyawan secara manual Sebelumnya nilai KPI hanya bisa diubah lewat kalkulasi otomatis sistem. Sekarang admin dapat mengisi nilai aktual, status, performance level, dan catatan langsung dari halaman KPI karyawan. KPIController.php, kpi/admin-edit.blade.php, kpi/show.blade.php
2 BARU Audit logging bisa diaktifkan / dinonaktifkan Ditambahkan tombol ON/OFF di halaman Audit Trail. Saat dimatikan (OFF), sistem tidak akan mencatat aktivitas apapun ke dalam log. AuditController.php, AuditLog.php
3 BARU Filter log berdasarkan tanggal dan tipe event Admin sekarang bisa menyaring tampilan log berdasarkan rentang tanggal (dari-sampai) dan tipe event (Created, Updated, Deleted). AuditController.php
4 BARU Hapus (purge) audit log sebelum tanggal tertentu Admin dapat menghapus semua log lama sebelum tanggal yang dipilih secara permanen dari database melalui tombol "Purge Log". AuditController.php, audit/index.blade.php
5 BARU Perintah otomatis bersihkan log lama (Artisan Command) Dibuat perintah: php artisan audit:purge --days=90 yang bisa dijadwalkan untuk menghapus log otomatis setiap periode tertentu tanpa perlu masuk ke aplikasi. Commands/PurgeAuditLogs.php
6 BARU Sidebar bisa dibuka / ditutup di layar desktop Ditambahkan tombol toggle di pojok kiri atas halaman. Saat diklik, sidebar akan menyempit atau melebar. Status ini diingat oleh browser sehingga tidak berubah saat halaman di-refresh. layouts/dashboard.blade.php
7 UI Redesign halaman Audit Trail — enterprise grade Tampilan halaman audit diubah total: hero header dengan gradien, kartu statistik, badge warna per tipe event, avatar inisial pengguna, modal diff BEFORE/AFTER yang jelas, responsif di semua ukuran layar. Sekaligus memperbaiki bug XSS di kolom User. audit/index.blade.php
